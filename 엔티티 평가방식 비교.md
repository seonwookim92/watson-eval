⏺ 엔티티 추출 평가 방식 비교                                                                                                                                             
                                                                                                                                                                         
  평가 목적: 모델이 추출한 엔티티 목록이 Ground Truth와 얼마나 일치하는지 측정.                                                                                          
   
  ---                                                                                                                                                                    
  예시 데이터                                                     
             
  Ground Truth (GT)
  ["CVE-2021-44228", "Apache Log4j", "remote code execution", "threat actor"]                                                                                            
                                                                             
  Predicted (Pred)                                                                                                                                                       
  ["CVE-2021-44228", "Log4j library", "RCE vulnerability", "attacker group", "Windows"]                                                                                  
                                                                                                                                                                         
  ---                                                                                                                                                                    
  1. Jaccard (토큰 집합 겹침)                                     
                                                                                                                                                                         
  각 문자열을 토큰(단어) 집합으로 변환 후 겹치는 비율로 유사도를 계산. Greedy matching으로 최적 쌍 구성.
                                                                                                                                                                         
  similarity(a, b) = |tokens(a) ∩ tokens(b)| / |tokens(a) ∪ tokens(b)|                                                                                                   
                                                                                                                                                                         
  ┌─────────────────────┬─────────────────────────┬────────────────────┬────────────────────────┐                                                                        
  │        Pred         │           GT            │    토큰 교집합     │         유사도         │                                                                        
  ├─────────────────────┼─────────────────────────┼────────────────────┼────────────────────────┤                                                                        
  │ "CVE-2021-44228"    │ "CVE-2021-44228"        │ {CVE, 2021, 44228} │ 1.00 ✓                 │
  ├─────────────────────┼─────────────────────────┼────────────────────┼────────────────────────┤
  │ "Log4j library"     │ "Apache Log4j"          │ {log4j}            │ 0.33 ✓ (threshold=0.2) │                                                                        
  ├─────────────────────┼─────────────────────────┼────────────────────┼────────────────────────┤                                                                        
  │ "RCE vulnerability" │ "remote code execution" │ {}                 │ 0.00 ✗                 │                                                                        
  ├─────────────────────┼─────────────────────────┼────────────────────┼────────────────────────┤                                                                        
  │ "attacker group"    │ "threat actor"          │ {}                 │ 0.00 ✗                 │
  ├─────────────────────┼─────────────────────────┼────────────────────┼────────────────────────┤                                                                        
  │ "Windows"           │ —                       │ —                  │ FP                     │
  └─────────────────────┴─────────────────────────┴────────────────────┴────────────────────────┘                                                                        
                                                                  
  TP=2, FP=3, FN=2  →  F1 = 2·2/(2·2+3+2) = 0.44                                                                                                                         
                                                                                                                                                                         
  특징: 빠르고 결정론적. 표현이 다르면 못 잡음 ("RCE" ↔ "remote code execution").                                                                                        
                                                                                                                                                                         
  ---                                                                                                                                                                    
  2. Embedding (의미 벡터 유사도)                                 
                                                                                                                                                                         
  각 문자열을 임베딩 벡터로 변환 후 코사인 유사도 계산. Greedy matching으로 최적 쌍 구성.
                                                                                                                                                                         
  similarity(a, b) = cos(embed(a), embed(b))                      
                                                                                                                                                                         
  ┌─────────────────────┬─────────────────────────┬───────────────┬─────────────────────────┐                                                                            
  │        Pred         │           GT            │ 코사인 유사도 │          결과           │                                                                            
  ├─────────────────────┼─────────────────────────┼───────────────┼─────────────────────────┤                                                                            
  │ "CVE-2021-44228"    │ "CVE-2021-44228"        │ 1.00          │ ✓                       │
  ├─────────────────────┼─────────────────────────┼───────────────┼─────────────────────────┤
  │ "Log4j library"     │ "Apache Log4j"          │ 0.82          │ ✓                       │                                                                            
  ├─────────────────────┼─────────────────────────┼───────────────┼─────────────────────────┤                                                                            
  │ "RCE vulnerability" │ "remote code execution" │ 0.78          │ ✓ ← Jaccard 못 잡은 것  │                                                                            
  ├─────────────────────┼─────────────────────────┼───────────────┼─────────────────────────┤                                                                            
  │ "attacker group"    │ "threat actor"          │ 0.71          │ ✗ (threshold=0.75 미만) │
  ├─────────────────────┼─────────────────────────┼───────────────┼─────────────────────────┤                                                                            
  │ "Windows"           │ —                       │ —             │ FP                      │
  └─────────────────────┴─────────────────────────┴───────────────┴─────────────────────────┘                                                                            
                                                                  
  TP=3, FP=2, FN=1  →  F1 = 2·3/(2·3+2+1) = 0.67
                                                                                                                                                                         
  특징: 의미적 유사성 포착. 약어·패러프레이즈에 강함. 임베딩 모델 품질에 의존.                                                                                           
                                                                                                                                                                         
  ---                                                                                                                                                                    
  3. LLM-as-a-Judge (의미 판단)                                   
                                                                                                                                                                         
  LLM에게 두 엔티티 목록을 주고 "같은 실세계 개체를 가리키는가?" 판단 요청. 배치로 처리.
                                                                                                                                                                         
  프롬프트: 아래 Predicted와 Ground Truth 엔티티 쌍 중            
            동일한 실세계 개체를 가리키는 쌍을 골라라.                                                                                                                   
                                                                                                                                                                         
  ┌─────────────────────┬─────────────────────────┬──────────┬──────────────────────────────────┐                                                                        
  │        Pred         │           GT            │ LLM 판단 │               이유               │                                                                        
  ├─────────────────────┼─────────────────────────┼──────────┼──────────────────────────────────┤                                                                        
  │ "CVE-2021-44228"    │ "CVE-2021-44228"        │ ✓        │ 동일 CVE ID                      │
  ├─────────────────────┼─────────────────────────┼──────────┼──────────────────────────────────┤                                                                        
  │ "Log4j library"     │ "Apache Log4j"          │ ✓        │ 같은 소프트웨어                  │                                                                        
  ├─────────────────────┼─────────────────────────┼──────────┼──────────────────────────────────┤                                                                        
  │ "RCE vulnerability" │ "remote code execution" │ ✓        │ RCE = remote code execution      │                                                                        
  ├─────────────────────┼─────────────────────────┼──────────┼──────────────────────────────────┤                                                                        
  │ "attacker group"    │ "threat actor"          │ ✓        │ 문맥상 동의어 ← EMB도 못 잡은 것 │
  ├─────────────────────┼─────────────────────────┼──────────┼──────────────────────────────────┤                                                                        
  │ "Windows"           │ —                       │ FP       │ GT에 대응 없음                   │
  └─────────────────────┴─────────────────────────┴──────────┴──────────────────────────────────┘                                                                        
                                                                  
  TP=4, FP=1, FN=0  →  F1 = 2·4/(2·4+1+0) = 0.89                                                                                                                         
                                                                                                                                                                         
  특징: 가장 정확. 문맥·도메인 지식 활용. 느리고 비결정론적(재현성 낮음).                                                                                                
                                                                                                                                                                         
  ---                                                                                                                                                                    
  종합 비교                                                       
                                                                                                                                                                         
  ┌─────────────┬───────────────┬─────────────┬──────────────┐    
  │             │    Jaccard    │  Embedding  │     LLM      │                                                                                                           
  ├─────────────┼───────────────┼─────────────┼──────────────┤
  │ 속도        │ 매우 빠름     │ 빠름        │ 느림         │                                                                                                           
  ├─────────────┼───────────────┼─────────────┼──────────────┤    
  │ 재현성      │ 완전 결정론적 │ 결정론적    │ 비결정론적   │                                                                                                           
  ├─────────────┼───────────────┼─────────────┼──────────────┤                                                                                                           
  │ 의미 이해   │ ✗             │ 부분적      │ ✓            │                                                                                                           
  ├─────────────┼───────────────┼─────────────┼──────────────┤                                                                                                           
  │ 약어 처리   │ ✗             │ △           │ ✓            │    
  ├─────────────┼───────────────┼─────────────┼──────────────┤                                                                                                           
  │ 동의어 처리 │ ✗             │ △           │ ✓            │    
  ├─────────────┼───────────────┼─────────────┼──────────────┤                                                                                                           
  │ 비용        │ 없음          │ 임베딩 서버 │ LLM API 호출 │    
  ├─────────────┼───────────────┼─────────────┼──────────────┤                                                                                                           
  │ F1 (예시)   │ 0.44          │ 0.67        │ 0.89         │    
  └─────────────┴───────────────┴─────────────┴──────────────┘                                                                                                           
                                                                  
  세 방식을 모두 보고하는 이유: Jaccard는 재현 가능한 하한선, Embedding은 중간 기준, LLM은 상한선 역할을 해서 모델 성능의 범위를 보여줌. 또한 LLM 판단 결과를 Type       
  Matching의 입력으로 재사용하기 때문에 LLM 매칭이 핵심 기준이 됨.
